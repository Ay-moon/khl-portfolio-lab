Rôles concernés

TRADER exécute les ordres. ASSET_MANAGER prend les décisions d'allocation. RISK_ANALYST surveille les limites avant et après exécution. QUANT lance les simulations automatisées.

Comment passe-t-on un ordre sur cette plateforme ?
Mode actuel : Les ordres sont générés automatiquement par le moteur de simulation (run_mvp.py via le bouton Simulation Trading (60j) dans le Monitoring). Il n'y a pas (encore) de saisie manuelle d'ordre dans l'interface.
Décision (Asset Manager)

L'Asset Manager décide d'acheter ou vendre un titre selon l'analyse de performance et les signaux du module Quant (backtesting, momentum).

Exécution (Trader)

Le Trader lance la simulation depuis Monitoring → Simulation Trading (60j). Le moteur calcule automatiquement les ordres selon la stratégie de rotation configurée.

Contrôle (Risk Analyst)

Le Risk Analyst vérifie que la VaR, le drawdown et les limites de concentration sont respectés après exécution dans le module Risk Management.

Calcul des frais et du slippage
Formule d'exécution BUY
execution_price = market_price × (1 + slippage_bps/10000) gross_amount = quantity × execution_price fee_amount = gross_amount × fee_bps/10000 net_cash_flow = -(gross_amount + fee_amount) # Paramètres par défaut fee_bps = 5 → 0.05% slippage_bps = 3 → 0.03%
Formule d'exécution SELL
execution_price = market_price × (1 - slippage_bps/10000) gross_amount = quantity × execution_price fee_amount = gross_amount × fee_bps/10000 net_cash_flow = gross_amount - fee_amount # Le slippage vient en déduction du prix de vente # (impact de marché négatif = moins bon prix obtenu)
Source : trading-sim/engine/simulator.py — méthode TradingSimulator.execute_market_order()
Stratégie de Rotation (SIM_MVP)
Algorithme

La stratégie RotationStrategy (trading-sim/strategies/rotation_strategy.py) tourne sur 3 actifs : AAPL, MSFT, SPY. Toutes les 10 sessions, elle fait une rotation : vend le plus performant et achète le moins performant. C'est un momentum inversé simple pour générer des données de test réalistes.

CommandoQuant — Pricing & VaR Dérivés (TRADER / QUANT / ADMIN)
Macro Excel VBA connectée à SQL Server

CommandoQuant.xlsm est une salle des marchés VBA pour le pricing d'options et le calcul de VaR dérivés. Elle se lance depuis Trading Desk → Ouvrir dans Excel (les rôles TRADER, QUANT et ADMIN y ont accès).
