Créer un portefeuille
Rôles concernés

ASSET_MANAGER, QUANT, ADMIN peuvent créer des portefeuilles. Le wizard guide l'utilisateur en 6 étapes avant l'instanciation en base de données.

Les 6 Étapes du Wizard
#ÉtapeDonnées saisiesValidation
1 Identité Code portefeuille (ex: PF001), Nom, Devise (EUR/USD/GBP) Code unique dans DimPortfolio
2 Univers investissable Sélection des instruments STOOQ (actions, indices, ETF, FX) Instruments présents dans stg_bourso_price_history
3 Profil risque CONSERVATIVE / BALANCED / AGGRESSIVE Stocké en session jusqu'à confirmation
4 Stratégie d'allocation Equal-weight / Risk-parity / Momentum / Personnalisé Détermine l'algorithme de simulation
5 Paramètres de simulation Capital initial, Frais (bps), Date de départ Utilisés par le moteur run_mvp.py
6 Contraintes réglementaires Limites VaR, concentration max par actif, tracking error Contrôlées par le module Risk
Ce qui est créé en base SQL à la confirmation
Déclencheur

À l'étape de confirmation (bouton Créer le portefeuille), la fonction _create_portfolio() dans portfolio/routes.py est appelée.

Table SQLColonne(s) insérée(s)Valeur
dbo.DimPortfolio PortfolioCode Code saisi à l'étape 1 (ex: PF001)
PortfolioName Nom descriptif saisi
BaseCurrency EUR / USD / GBP
RiskProfile CONSERVATIVE / BALANCED / AGGRESSIVE
InceptionDate Date de départ de la simulation
IsActive 1 (par défaut)
dbo.AppLog Module portfolio Action CREATE PORTFOLIO PF001 — traçabilité de la création
Simulation associée — ce qui se passe après
Lancement du moteur de simulation

Script trading-sim/engine/run_mvp.py lancé depuis le Monitoring. Il génère des trades simulés sur les instruments sélectionnés.

Insertion dans FactTrades

Chaque ordre simulé (BUY/SELL) est enregistré dans dbo.FactTrades avec prix, quantité, frais et slippage.

Calcul des positions journalières

dbo.PortfolioPositionsDaily — positions nettes, valeur de marché, PnL latent, poids par actif.

Calcul Performance & Risk

Script analytics/performance_risk_mvp.py : calcule NAV, PnL cumulé, VaR 95%, Sharpe ratio, drawdown. Stocké dans PortfolioPnLDaily et RiskMetricsDaily.

Schéma SQL impliqué
DimPortfolio (créé à la confirmation du wizard) └── PortfolioKey (PK, IDENTITY) └── PortfolioCode : "PF001" └── PortfolioName : "Mon portefeuille CIB" └── BaseCurrency : "EUR" └── RiskProfile : "BALANCED" └── InceptionDate : 2026-01-05 └── IsActive : 1 │ ├── FactTrades (trades simulés par run_mvp.py) │ └── TradeDateKey → DimDate.DateKey │ └── PortfolioKey → DimPortfolio.PortfolioKey │ └── SecurityKey → DimSecurity.SecurityKey │ └── Side : "BUY" | "SELL" │ └── Quantity, Price, FeeAmount, SlippageAmount │ ├── PortfolioPositionsDaily (snapshot quotidien) │ └── Quantity, AvgCost, MarketValue │ └── UnrealizedPnL, WeightPct │ ├── PortfolioPnLDaily (performance) │ └── Nav, DailyPnL, CumPnL, ReturnPct │ └── RiskMetricsDaily (risque) └── Volatility20d, MaxDrawdown, VaR95, SharpeRatio, Beta
