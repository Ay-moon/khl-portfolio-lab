01 — Vue d'ensemble

Ce que fait la plateforme, pour qui, et pourquoi elle est construite ainsi.

🏦
KHL Bank CIB Platform est une plateforme de formation et de démonstration simulant l'environnement d'une banque d'investissement (CIB). Elle reproduit les flux réels : décision d'allocation → exécution d'ordres → comptabilisation → contrôle du risque → reporting. Le cloisonnement des tâches (Front / Middle / Back Office) y est appliqué avec la même rigueur qu'en production.

04 — Flask & Blueprints

Comment l'application est assemblée au démarrage et comment les requêtes sont routées.

⚡
Démarrage : python webapp/app.py → init_db() (crée les tables si absentes) → enregistrement des 10 blueprints → serveur Flask sur http://localhost:5000. Un seul process, pas de build step, pas de webpack.
Cycle d'une requête HTTP
1
Requête HTTP
GET /trading/ ou POST /trading/order/new
2
before_request
load_user() → g.user depuis session Flask
3
Décorateurs
@login_required → vérifie session["user_id"] · @role_required → vérifie session["role"]
4
Route handler
Logique métier + requêtes SQL via with db.db_cursor() as cur:
5
render_template
Jinja2 → HTML avec variables + context_processor global (roles, modules)
6
Réponse
200 OK / 302 Redirect / 403 access_denied.html

08 — Base de données SQL Server

3 bases distinctes sur le même serveur. Authentification Windows uniquement — aucun mot de passe SQL.

SmartAssetAdvicedb (Gold)

Base principale. Tables Gold (DimPortfolio, FactTrades...), tables applicatives (AppUsers, AppLog, AppSettings) et tables CIB (comptabilité, settlement, réconciliation). Toutes créées par init_db() au démarrage.

KHLWorldInvest (Staging)

Cotations historiques Boursorama/STOOQ. Table stg.stg_bourso_price_history. Alimente FactPrice via le script load_factprice_from_stg.py.

CommandoQuant (Dérivés)

Dédiée au pricing options. Contient tbl_Greeks. Alimentée par le VBA Excel. Lue par var_engine.py via computed columns ImpliedVol / Price.
